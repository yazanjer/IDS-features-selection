# Deliberately minimal pins. Colab ships compatible versions of
# numpy / pandas / scipy / scikit-learn / matplotlib / seaborn / tqdm /
# joblib already, and ANY pin we add gets pip into a resolver tangle
# (e.g. pandas==2.2.2 caps numpy at 2.0.2 but Colab's scipy wheel was
# built against numpy 2.1+, so scipy then fails to import with
# "cannot import name '_center' from numpy._core.umath"). Leave Colab's
# preinstalled versions alone and only declare what we actually need
# installed or upgraded.
#
# numpy>=2.2 floor: Colab 2026.04 ships numpy 2.0.2 BUT its base image has
# a partially-updated numpy/testing/_private/utils.py that calls
# np._core._multiarray_umath._blas_supports_fpe — a symbol added in
# numpy 2.2.0. sklearn imports numpy.testing transitively, so the import
# crashes with AttributeError unless numpy is upgraded to >=2.2. Lower
# bound only — no upper bound, no exact pin — per the v3 rule.
numpy>=2.2
pandas
scipy
scikit-learn
imbalanced-learn
matplotlib
seaborn
tqdm
joblib
